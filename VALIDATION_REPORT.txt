PWA VALIDATION
--------------
Manifest, icons, service worker, offline asset cache and GitHub Pages workflow
are included. HTTPS or localhost is required for installation and service-worker
features.

KNOWN ADAPTATIONS
-----------------
The hardware display, GPIO, potentiometer, EEPROM, LittleFS, SD and I2S layers are
replaced by browser canvases, touch controls, IndexedDB and HTML audio.

The content databases, stage schedule, core care simulation, world systems and
all named games are ported. Several Pico minigames use phone-friendly versions
of their original mechanics rather than line-for-line Arduino execution.

A serverless PWA cannot guarantee exact notification timing while fully closed.
Elapsed time is always processed when the app reopens.

RUNTIME LIMITATION
------------------
The container used for packaging could not complete a Chromium headless runtime
capture because the sandboxed browser process did not start successfully. The
project received syntax, data, manifest and archive validation, but should still
be tested on the target Android browser after deployment.
